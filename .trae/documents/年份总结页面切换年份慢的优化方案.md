# 年份总结页面切换年份慢的优化方案

## 问题分析

经过分析，年份总结页面切换年份时数据加载慢的主要原因是 `fetchYearlySummary` 函数存在**n+1查询问题**：

1. 函数首先获取年度总结记录（1个请求）
2. 然后获取重要事件（1个请求）
3. 接着获取AI分析部分（1个请求）
4. **问题所在**：对于每个AI分析部分，函数会**单独发起一个请求**获取其关联的观点（n个请求，n为AI分析部分的数量）
5. 最后获取投资图片（1个请求）

总计：n+4个请求，当有多个AI分析部分时，会导致大量网络请求，从而导致数据加载缓慢。

## 优化方案

使用Supabase的**嵌套查询功能**（关系查询），一次性获取所有数据，减少数据库请求次数。

### 优化后的查询流程

1. 获取年度总结记录（1个请求）
2. **并行执行**以下查询：
   - 获取重要事件（1个请求）
   - **一次性获取所有AI分析部分及其关联的观点**（1个请求，使用嵌套查询）
   - 获取投资图片（1个请求）

总计：4个请求，与AI分析部分的数量无关，大幅减少网络请求次数。

## 优化实现

### 修改 `fetchYearlySummary` 函数

将当前的串行请求改为并行请求，并使用Supabase的嵌套查询获取AI分析部分及其观点。

```typescript
export async function fetchYearlySummary(year: string): Promise<YearlySummary | null> {
  try {
    // 1. 获取年度总结记录
    const { data: summaryData, error: summaryError } = await supabase
      .from('yearly_summaries')
      .select('id')
      .eq('year', year)
      .single()

    if (summaryError) {
      if (summaryError.code === 'PGRST116') {
        return null
      }
      throw summaryError
    }

    const summaryId = summaryData.id

    // 2. 并行获取所有数据
    const [eventsResult, aiResult, imagesResult] = await Promise.all([
      // 获取重要事件
      supabase
        .from('important_events')
        .select('*')
        .eq('yearly_summary_id', summaryId)
        .order('start_date', { ascending: true }),
      
      // 使用嵌套查询一次性获取AI分析部分及其观点
      supabase
        .from('ai_analysis_sections')
        .select('* , ai_analysis_opinions(*)', { count: 'exact' })
        .eq('yearly_summary_id', summaryId),
      
      // 获取投资图片
      supabase
        .from('investment_images')
        .select('*')
        .eq('yearly_summary_id', summaryId)
        .order('created_at', { ascending: true })
    ])

    // 处理结果
    if (eventsResult.error) throw eventsResult.error
    if (aiResult.error) throw aiResult.error
    if (imagesResult.error) throw imagesResult.error

    // 构建AI分析数据
    const aiAnalyses: AIAnalysisSection[] = aiResult.data.map((section: any) => ({
      id: section.id,
      title: section.title,
      content: section.content,
      opinions: section.ai_analysis_opinions.map((opinion: any) => ({
        id: opinion.id,
        content: opinion.content,
        analysis: opinion.analysis
      }))
    }))

    // 构建返回数据
    return {
      year,
      importantEvents: eventsResult.data.map(convertFromSupabaseImportantEvent),
      aiAnalyses,
      investmentImages: imagesResult.data.map(convertFromSupabaseInvestmentImage)
    }
  } catch (error) {
    console.error('Error fetching yearly summary:', error)
    return null
  }
}
```

## 预期效果

1. **大幅减少网络请求次数**：从原来的n+4个请求减少到4个请求
2. **提高数据加载速度**：并行请求比串行请求更快
3. **优化用户体验**：年份切换时数据加载更流畅
4. **更好的可扩展性**：即使有大量AI分析部分，性能也不会明显下降

- 使用Supabase的**嵌套查询**功能：`select('* , related_table(*)`

- 避免"n+1查询问题"
- 优化网络请求数量
